/**
 * Shared HitPay order processing logic.
 *
 * Called by:
 *   - /api/webhooks/hitpay  (normal path, webhook fires after payment)
 *   - /api/payments/hitpay/status  (fallback path, webhook missed — HitPay API confirms payment)
 *
 * Takes a completed pending payment row + payment details and:
 *   1. Creates the order via create_order_atomic RPC
 *   2. Marks hitpay_pending_payments as completed
 *   3. Claims promo + referral
 *   4. Sends receipt (best-effort with retry)
 *   5. Runs compliance check for high-value orders
 */
